# KOL 招募自动化系统操作手册

## 1. 系统目标

本系统用于把 KOL 招募流程标准化、自动化：

`KOL 填表 -> 自动入库 -> 自动评分 -> 自动分层 -> 自动生成联系话术 -> 跟进提醒 -> 数据看板`

适合以下场景：

- 校园 KOL 招募
- 内容创作者招募
- 达人合作筛选
- 新产品冷启动推广
- 注册拉新和任务制合作

## 2. 客户需要准备什么

安装前请准备：

- Windows 电脑一台。
- Node.js 20 LTS 或 22 LTS。
- 一个 Google Sheet。
- 一个 Google Cloud 服务账号 JSON。
- Google Sheets API 已启用。
- 一个可用的 SMTP 邮箱和应用专用密码。

## 3. Google 服务账号准备

### 3.1 创建 Google Cloud 项目

进入 Google Cloud Console，创建或选择一个项目。

### 3.2 启用 Google Sheets API

在 Google Cloud Console 中启用：

`Google Sheets API`

如果没有启用，系统会在访问表格时返回类似：

`Google Sheets API has not been used or is disabled`

### 3.3 创建服务账号

进入 IAM & Admin -> Service Accounts，创建服务账号。

创建完成后，下载 JSON key。这个 JSON 文件只给安装器使用，不要发给无关人员。

### 3.4 共享 Google Sheet

打开目标 Google Sheet，点击“共享”，把服务账号 JSON 里的 `client_email` 加进去，权限设为“编辑者”。

服务账号邮箱通常长这样：

`xxx@xxx.iam.gserviceaccount.com`

## 4. 一键安装

解压安装包后，双击：

`一键安装配置.cmd`

安装器会依次询问：

- 公司/项目名称
- Google Sheet 链接或 Sheet ID
- Google 服务账号 JSON 文件路径
- 发件邮箱地址
- SMTP 服务器
- SMTP 端口
- 是否 SSL/TLS
- SMTP 应用专用密码
- 收件提醒邮箱
- 运营负责人邮箱

安装器会自动完成：

- 安装 n8n
- 创建本地 n8n 数据目录
- 导入 Google Sheets 凭证
- 导入 SMTP 邮箱凭证
- 初始化 Google Sheet 表结构
- 生成客户专属工作流
- 导入并激活工作流
- 启动 n8n

安装完成后打开：

`http://127.0.0.1:5678`

## 5. Google Sheet 表结构

安装器会自动创建以下工作表：

- `Form Responses`
- `KOL Database`
- `Follow-up Tracker`
- `Scoring Rules`
- `Dashboard`

### 5.1 Form Responses

这是 KOL 提交入口。核心字段包括：

- 提交时间
- 姓名
- 联系邮箱
- 联系方式
- 学校
- 年级/身份
- 地区
- 主要社交平台
- 社交平台链接
- 粉丝数
- 平均播放量
- 平均互动量
- 内容类型
- 受众画像
- 合作意向
- 报价
- 可接受合作方式
- 过往合作经验
- 代表合作案例
- 是否同意后续联系
- n8n处理状态

`n8n处理状态` 为空时，系统会认为这是待处理的新提交。处理完成后会回写为 `已处理`。

### 5.2 KOL Database

系统会把有效提交写入这里，并自动生成：

- KOL ID
- 各项评分
- 总分
- KOL分层
- 推荐合作方式
- 自动联系话术
- 当前跟进状态
- 下一次跟进日期
- 注册数
- 任务完成数

### 5.3 Follow-up Tracker

用于跟踪跟进动作，包括：

- 跟进ID
- KOL ID
- 姓名
- KOL分层
- 跟进方式
- 跟进内容
- 下一步动作
- 下一次跟进日期
- 是否需要提醒

### 5.4 Dashboard

看板追踪：

- 提交人数
- 有效KOL数
- A/B/C 类 KOL 数
- 联系成功率
- 合作转化率
- 注册数合计
- 任务完成数合计

## 6. 正式招募流程

### 6.1 推荐方式：Google Form

1. 创建 Google Form。
2. 按 `Form Responses` 的字段创建问题。
3. 在 Google Form 的“回复”里绑定到安装器初始化过的 Google Sheet。
4. 确保回复进入 `Form Responses` 表。
5. 发布表单链接给 KOL。

### 6.2 临时方式：手动填表

如果暂时没有 Google Form，可以直接在 `Form Responses` 新增一行，填写 KOL 信息，并保持 `n8n处理状态` 为空。

n8n 每 5 分钟自动扫描一次。

## 7. 自动评分规则

系统按 100 分制评分：

- 粉丝质量：20 分
- 内容匹配度：25 分
- 互动率：20 分
- 校园影响力：15 分
- 报价合理性：10 分
- 过往合作表现：10 分

分层规则：

- A类：总分 >= 80
- B类：总分 >= 60 且 < 80
- C类：总分 < 60

## 8. 日常运营怎么用

运营人员每天主要看：

- `KOL Database`：优先处理 A 类和 B 类。
- `Follow-up Tracker`：查看今天该联系谁。
- `Dashboard`：查看总体招募效果。

建议状态流转：

- 待联系
- 已联系
- 已回复
- 已报价
- 合作中
- 已合作
- 暂缓
- 拒绝
- 观察池

## 9. n8n 工作流说明

### Form Intake Automation

每 5 分钟执行一次：

1. 读取 `Form Responses`
2. 找出 `n8n处理状态` 为空的行
3. 自动评分
4. 自动分层
5. 生成联系话术
6. 写入 `KOL Database`
7. 写入 `Follow-up Tracker`
8. 回写原始行 `n8n处理状态 = 已处理`
9. 尝试发送邮件提醒

### Daily Follow-up Reminder

每天 10 点执行：

1. 读取 `KOL Database`
2. 找出到期且仍需跟进的 KOL
3. 生成跟进提醒邮件

### Weekly Dashboard Report

每周五 18 点执行：

1. 读取 `KOL Database`
2. 汇总核心指标
3. 生成周报邮件

## 10. 常见问题

### 为什么 KOL 没有被处理？

检查：

- 是否填在 `Form Responses`
- 是否有姓名和社交平台链接
- `n8n处理状态` 是否为空
- n8n 是否正在运行
- 主工作流是否处于 Active

### 为什么重复入库？

正常情况下不会重复。系统会回写 `已处理`。如果手动把 `n8n处理状态` 清空，系统会重新处理。

### 邮件没发出去怎么办？

见 `邮件提醒功能说明.md`。

### 能不能发给其他公司用？

可以，但只能发原始安装包。不要把安装后生成的 `runtime`、`config.local.json`、`logs` 打包给别人。
