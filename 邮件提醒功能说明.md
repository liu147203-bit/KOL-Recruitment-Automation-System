# 邮件提醒功能说明

## 1. 邮件提醒负责什么

系统中有三类邮件提醒：

- 新 KOL 提交评分完成后，给运营负责人发邮件。
- 每天 10 点，发送当天需要跟进的 KOL 列表。
- 每周五 18 点，发送 KOL 招募数据周报。

邮件内容包括：

- KOL 基本信息
- 自动评分
- A/B/C 分层
- 推荐合作方式
- 自动联系话术
- 下一次跟进日期
- 看板摘要

## 2. 邮件功能是否已经做好

邮件功能已经写入 n8n 工作流，并且 SMTP 凭证会由安装器自动导入。

如果 SMTP 服务器、端口、账号、应用专用密码正确，并且电脑所在网络允许访问 SMTP 端口，邮件会自动发送。

## 3. 为什么有时邮件发不出去

常见原因不是系统逻辑问题，而是网络或邮箱策略问题：

- 公司网络屏蔽 SMTP 465/587 端口。
- 电脑所在地区或网络无法访问 Gmail SMTP。
- SMTP 应用专用密码错误。
- 邮箱没有开启 SMTP/应用专用密码。
- 安全软件或代理阻止了 n8n 连接外部 SMTP。

典型错误：

- `queryA ETIMEOUT smtp.gmail.com`
- `connect ETIMEDOUT`
- `ECONNECTION`
- `Invalid login`
- `Authentication failed`

其中 `queryA ETIMEOUT smtp.gmail.com` 通常表示当前网络/DNS 无法稳定访问 Gmail SMTP。

## 4. 邮件失败会不会影响主流程

不会。

工作流已设置为：邮件发送失败不阻断主流程。

也就是说，即使 SMTP 暂时不可用，以下功能仍然正常：

- KOL 自动入库
- 自动评分
- 自动分层
- 自动生成联系话术
- 写入跟进表
- 回写 `已处理`
- Dashboard 看板统计

邮件恢复后，后续提醒会自动恢复发送。

## 5. Gmail SMTP 推荐配置

如果使用 Gmail：

- SMTP 服务器：`smtp.gmail.com`
- 端口：`465`
- SSL/TLS：`true`
- 用户名：完整 Gmail 地址
- 密码：Gmail 应用专用密码，不是网页登录密码

如果 465 被网络屏蔽，可以尝试：

- SMTP 服务器：`smtp.gmail.com`
- 端口：`587`
- SSL/TLS：`false`
- STARTTLS：启用

当前安装脚本默认按 465 + SSL/TLS 导入。若需要改成 587，可在 n8n 的 Credentials 里编辑 `KOL SMTP` 凭证。

## 6. 如何测试 SMTP 端口是否可用

在 Windows PowerShell 中执行：

```powershell
Test-NetConnection smtp.gmail.com -Port 465
Test-NetConnection smtp.gmail.com -Port 587
```

如果结果中：

`TcpTestSucceeded : True`

说明端口可用。

如果是：

`TcpTestSucceeded : False`

说明当前网络无法连接该端口，需要换网络、换 SMTP 服务商，或让 IT 放行端口。

## 7. 如何在 n8n 里查看邮件节点

打开：

`http://127.0.0.1:5678`

进入工作流：

- `公司名 KOL - Form Intake Automation`
- `公司名 KOL - Daily Follow-up Reminder`
- `公司名 KOL - Weekly Dashboard Report`

找到 `Email` 节点，点击查看执行结果。

如果邮件节点输出里只有 `error` 字段，但整个工作流显示成功，这是正常设计：邮件失败不阻断主流程。

## 8. 如何确认邮件功能逻辑可用

满足以下条件即可确认邮件功能逻辑已正确接入：

- n8n 中存在 `KOL SMTP` 凭证。
- 工作流里的 Email 节点使用 `KOL SMTP`。
- Email 节点的 From/To 为安装时填写的邮箱。
- 工作流执行时已经走到 Email 节点。
- 若网络不可达，Email 节点返回 SMTP/DNS/timeout 错误，而不是字段缺失或凭证缺失错误。

这表示系统逻辑是通的，只差 SMTP 网络连通性或邮箱侧授权。

## 9. 推荐替代方案

如果 Gmail SMTP 长期不可用，可以改用：

- 企业邮箱 SMTP
- Outlook SMTP
- 163/QQ 企业邮箱 SMTP
- SendGrid
- Mailgun
- Amazon SES

只要对方提供 SMTP 服务器、端口、账号和授权码，就可以在 n8n 的 `KOL SMTP` 凭证中替换。
